Nodes descriptions

  • ELEMENT_NODE : most simple. Just an element like p and h1, divs etc

  • TEXT_NODE: The actual text. No React a gente costuma chamar de "children". A definição de children é mais ampla, mas é mais pra demonstrar que o texto dentro das tags é um TEXT_NODE.

  • CDATA_SECTION_NODE: "don't worry about them". "Character Data not parsed by the parser". OK!

  • PROCESSING_INSTRUCTION_NODE: Used in XML to embed application-specific instructions

  • COMMENT_NODE: dã.

  • DOCUMENT_NODE: the root document node

  • DOCUMENT_TYPE_NODE: Also simple. Just the <!DOCTYPE html> declaration

  • DOCUMENT_FRAGMENT_NODE: A DocumentFragment node is a lightweight document object. Sometimes you may want to extract a portion of the documents tree and create a new fragment.

The most common Nodes are:

  1. Document Node
  2. Element Nodes
  3. Text Nodes
  4. Coment Nodes
